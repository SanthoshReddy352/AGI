## What this file is

Historical freeze marker for Track 0–5. While the consolidation window was open, the project did not accept new layers, plugins, memory subsystems, routers, or workflow styles. Every PR in that window cited a Track ID (e.g. `Track 1.2: WorkingArtifact scope precedence`).

## Operating principles (in force during the freeze)

1. **No new layers until consolidation is done.** New components must replace existing ones, not run alongside.
2. **Every fix is a triple: change + retirement + test.** A PR isn't done until the old path is deleted and a cross-turn test guards the new behavior.
3. **"Wired" ≠ "done".** Definition of done: (a) new path is the only path, (b) old path deleted, (c) failing-without-fix test exists, (d) `docs/testing_guide.md` updated in same PR.
4. **One layer at a time.** Sequential consolidation tracks, never parallel.
5. **Cross-turn behavior tests are the acceptance gate.** Every UX bug being fixed survived 378+ unit tests — that bar is insufficient.
6. **Trace_id over hand-wavy logs.** Every routing/dispatch fix adds structured timing.
